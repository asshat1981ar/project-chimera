# Core-AI Production Wiring Guide

**Status:** ✅ Verified - Production swap ready

## Architecture

```
DialogueOrchestrator (singleton)
├── Primary: ProviderRouter (optional, injected if API keys present)
│   ├── GeminiAdapter
│   ├── GroqAdapter
│   └── OpenRouterAdapter
└── Fallback: FakeDialogueProvider (always available)
```

## Hilt Binding

**Module:** `app/src/main/kotlin/com/chimera/di/AiAdapterModule.kt`

```kotlin
@Provides
@Singleton
fun provideDialogueOrchestrator(
    fallback: FakeDialogueProvider,
    router: ProviderRouter
): DialogueOrchestrator {
    val orchestrator = DialogueOrchestrator(fallback)
    if (router.providers.isNotEmpty()) {
        orchestrator.setPrimaryProvider(router)
    }
    return orchestrator
}
```

## Production Swap Procedure

### Step 1: Add API Keys to `local.properties`

```properties
# AI Provider API Keys (optional)
GEMINI_API_KEY=your_gemini_key_here
GROQ_API_KEY=your_groq_key_here
OPENROUTER_API_KEY=your_openrouter_key_here
```

### Step 2: Verify BuildConfig Generation

```kotlin
// BuildConfig.java (generated)
public static final String GEMINI_API_KEY = "your_key";
public static final String GROQ_API_KEY = "your_key";
public static final String OPENROUTER_API_KEY = "your_key";
```

### Step 3: Provider Chain Activation

Providers activate automatically based on API key presence:
- If `BuildConfig.GEMINI_API_KEY.isNotBlank()` → Gemini added to chain
- If `BuildConfig.GROQ_API_KEY.isNotBlank()` → Groq added to chain
- If `BuildConfig.OPENROUTER_API_KEY.isNotBlank()` → OpenRouter added to chain

### Step 4: Verify at Runtime

```kotlin
// In DialogueSceneViewModel or debug screen
val isFallbackActive = orchestrator.isFallbackActive
// false = production AI active
// true = using FakeDialogueProvider templates
```

## Fallback Behavior

| Scenario | Behavior |
|----------|----------|
| No API keys | FakeDialogueProvider (authored templates) |
| API key present, provider healthy | Cloud provider used |
| Cloud provider throws exception | Falls back to FakeDialogueProvider |
| All cloud providers fail | Falls back to FakeDialogueProvider |

## Monitoring

Check `Logcat` for provider activity:
```
D/ProviderRouter: Turn generated by Gemini
W/ProviderRouter: Groq failed: timeout
W/DialogueOrchestrator: Primary provider failed, falling back
```

## Security Notes

- API keys stored in `local.properties` (gitignored)
- Never commit API keys to version control
- Use separate keys for dev/staging/production
- Rotate keys periodically
