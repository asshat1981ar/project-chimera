# Plan B — Codebase Analysis and Gap Identification

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a living codebase health report for Project Chimera that maps module completeness, integration wiring gaps, test coverage gaps, and architecture debt — grounded in current code, not planning docs.

**Architecture:** Two output artifacts: `docs/sdlc/codebase-health.md` (human-readable gap report) and `docs/sdlc/sprint-backlog.md` (machine-actionable item list in priority order). The health report is generated by the `sdlc-forge` agent against the current repo state. The backlog is derived from the health report and replaces the stale `task_plan.md` and `SPRINT-MANIFEST.md` claims.

**Tech Stack:** Gradle multi-module Android project (Kotlin, Jetpack Compose, Hilt, Room, Navigation Compose), `sdlc-forge` agent, git log, Detekt.

---

- Read: `SPRINT-MANIFEST.md`, `task_plan.md`, recent git log

### Task 2: Scan each module for completeness

For each module in the list below, run the grep and record findings.

**Files:**
- Read: every `src/main/java/**/*.kt` file in each module (via grep/glob)

- [ ] **Step 1: Identify unimplemented stubs in `chimera-core`**

Run:
```bash
grep -rn "TODO\|FIXME\|NotImplementedError\|error(\"not impl" chimera-core/src/ 2>/dev/null
```
Expected: zero hits (pure logic module — stubs here are blockers).

- [ ] **Step 2: Identify Hilt injection gaps in feature modules**

Run:
```bash
for mod in feature-home feature-map feature-dialogue feature-camp feature-journal feature-party feature-settings; do
  echo "=== $mod ==="
  grep -rn "@HiltViewModel\|@Inject" $mod/src/ 2>/dev/null | wc -l
done
```
Expected: non-zero counts for every module that has a ViewModel. Zero means the module has no injected ViewModels — flag it.

- [ ] **Step 3: Find unnavigated routes**

Run:
```bash
grep -rn "ChimeraRoutes\." app/src/ 2>/dev/null
```
Expected: at least one route constant per feature. Any feature module not referenced in nav routes is dead code.

- [ ] **Step 4: Locate DAO queries missing implementations**

Run:
```bash
grep -rn "@Query\|@Insert\|@Update\|@Delete" core-database/src/ 2>/dev/null | wc -l
```
Record count. Then:
```bash
grep -rn "TODO\|FIXME" core-database/src/ 2>/dev/null
```
Expected: zero TODOs in DAO files.

- [ ] **Step 5: Check domain use-case test coverage**

Run:
```bash
find domain/src/test -name "*Test*.kt" -o -name "*Spec*.kt" 2>/dev/null
```
Expected: at least one test file per use-case class. Missing test files = coverage gap.

Run:
```bash
ls domain/src/main/java/ -R 2>/dev/null | grep "UseCase\.kt"
```
Compare use-case count to test file count. Record difference.

- [ ] **Step 6: Identify missing Room migrations**

Run:
```bash
ls core-database/src/main/assets/databases/ 2>/dev/null || echo "no migration assets"
```
If migration assets exist, check schema version in entity annotations:
```bash
grep -rn "version\s*=" core-database/src/main/java/ 2>/dev/null | grep "@Database"
```

- [ ] **Step 7: Check `feature-journal` for any implementation**

Run:
```bash
find feature-journal/src -name "*.kt" 2>/dev/null | wc -l
```
Expected: if count is 0 or very low, `feature-journal` is a shell module. Flag as "skeleton — not started".
